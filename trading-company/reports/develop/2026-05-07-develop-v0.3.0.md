## Wijzigingen

### Wijziging 1 — Imbalance grootte-drempel (`_stap_3_imbalance`)

**Bestand:** `src/decision/decision_tree.py`

De functie `_stap_3_imbalance()` had twee parameters voor de `c670e7e` fix (`zone_binnen_bereik`) en heeft nu een derde: `min_size_pct` (doorgegeven vanuit `settings["strategy"]["imbalance_min_size_pct"]`).

**Logica na wijziging:**
- Blokkerende imbalances worden gesplitst op grootte: `grootte >= prijs * 0.0015` = significant, kleiner = micro
- Grote imbalances + zone binnen bereik → harde VETO (ongewijzigd)
- Micro-imbalances altijd → GEEL (score aftrek, geen stop)
- Grote imbalances + zone buiten bereik → GEEL (bestaand gedrag via `zone_binnen_bereik`)

**Nieuwe parameter in `settings.yaml`:**
```yaml
imbalance_min_size_pct: 0.0015   # 0.15% minimum range voor harde veto op imbalance
```

**Verwacht effect:** TAO (7 imbalances waarvan de meeste micro) en mogelijk BTC/SOL komen door stap 3 waar ze eerder hard blokten.

### Wijziging 2 — Zone nabijheid 2% → 3%

**Bestand:** `config/settings.yaml`

```yaml
zone_proximity_pct: 0.03      # was 0.02
```

Enkelvoudige parameterwijziging. HYPE stond op 2.3% van zijn EXTREME demand zone en werd geblokkeerd door de 2%-drempel. Bij 3% is HYPE bereikbaar.

## Technische details

### Merge baseline
Dev was 4 commits achter op main. Merge van main naar dev uitgevoerd vóór implementatie:
- `c670e7e` — zone_binnen_bereik fix (imbalance veto)
- `8792e11` — capabilities veld fix Paperclip API
- `6640b21` — process reports map fix
- `c921b61` — Process Agent toevoeging

### Parameteroverdracht
`min_size_pct` wordt doorgegeven vanuit `evaluate()` via `settings["strategy"].get("imbalance_min_size_pct", 0.0015)` — dezelfde patroon als `proximity_pct`. Default fallback zorgt voor backward compatibility als de parameter niet in `settings.yaml` staat.

### Interactie met `zone_binnen_bereik`
De twee condities zijn onafhankelijk:
- `zone_binnen_bereik = False` → altijd GEEL (bestaand gedrag)
- `zone_binnen_bereik = True` + alleen micro-imbalances → nieuw GEEL
- `zone_binnen_bereik = True` + grote imbalances → VETO (ongewijzigd)
